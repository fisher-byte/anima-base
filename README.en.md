## 📝 Documentation Standards

Each content file includes:
- ✅ YAML Frontmatter (source, date, tags)
- ✅ Core insights & framework summaries (not word-for-word transcripts)
- ✅ Verification status tags (verification_status)
- ✅ Table of contents
- ✅ Original source links
- ✅ Collection metadata

**Content Types**:
- `curated_summary`: Curated summary - Core insights and frameworks
- `full_transcript`: Full transcript - Complete text from podcasts, etc.
- `verified`: Verified - Fact-checked content

## 🌐 Bilingual Support

**Dual-Track Architecture**:

| Track | Location | Language | Description |
|-------|-----------|-----------|-------------|
| Original Files | `media/`, `files/` | Original language | MP3, PDF, EPUB, HTML - preserved as-is |
| Index Files | `people/{category}/{person}/` | English | Metadata and summaries in English |
| Translation Files | `translations/zh-cn/{category}/{person}/` | Chinese | Complete translations in Chinese |

**Cross-Linking**:
- English files link to Chinese: `**中文版** → [Chinese](../../translations/zh-cn/...)`
- Chinese files link to English: `**English Version** → [English](../../../people/...)`

**Translation Priority**:
1. Core framework documents (complete)
2. Expert profiles (complete)
3. Important podcast and article summaries (selective)
