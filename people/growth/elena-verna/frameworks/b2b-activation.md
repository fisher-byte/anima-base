# B2B Activation Framework

**来源**: Elena Verna, "I bet you are doing product activation all wrong" (2024)

---

## 框架概述

> "90% of the monetization and retention issues stem from incorrect activation."
> — Elena Verna

大多数 B2B 公司在激活指标上犯了根本性错误。这个框架帮助正确定义和衡量产品激活。

---

## 激活的正确定义

> "Activation is taking a user from **signing up** to **establishing a habit** around your core value prop."
> — Reforge Definition

**关键要素**:
- 从注册开始
- 到建立习惯为止
- 围绕核心价值主张

---

## 激活 vs 其他概念

| 概念 | 定义 | 与激活的关系 |
|------|------|--------------|
| **注册** | 创建账户 | 激活的起点 |
| **入职** | 完成初始设置 | 激活的一部分 |
| **Aha 时刻** | 首次体验价值 | 激活的里程碑 |
| **习惯** | 重复使用 | 激活的终点 |

---

## 常见激活错误

### 错误 1: 把入职当激活
- ❌ 用户完成了引导流程 = 激活
- ✅ 用户建立了使用习惯 = 激活

### 错误 2: 只看单一事件
- ❌ 用户执行了关键动作一次 = 激活
- ✅ 用户重复执行关键动作 = 激活

### 错误 3: 忽略时间窗口
- ❌ 任何时候达到标准 = 激活
- ✅ 在特定时间窗口内达到标准 = 激活

### 错误 4: 账户级 vs 用户级混淆
- B2B 需要同时关注个人用户激活和账户激活
- 两者的指标和标准不同

---

## 正确的激活框架

### Step 1: 识别核心价值行为
- 什么行为代表用户获得了核心价值？
- 这个行为是否可重复？

### Step 2: 定义习惯频率
- 用户应该多频繁使用这个功能？
- 什么频率代表"习惯"？

### Step 3: 设置时间窗口
- 用户应该在多长时间内达到激活？
- 第一周？第一月？

### Step 4: 区分用户级和账户级
- 个人用户的激活标准是什么？
- 账户整体的激活标准是什么？

## B2B 特殊考虑

### 双层激活模型

```
┌─────────────────────────────────────┐
│           账户激活                   │
│  (N个用户激活 + 跨部门使用)          │
│                                     │
│   ┌───────┐ ┌───────┐ ┌───────┐    │
│   │用户A  │ │用户B  │ │用户C  │    │
│   │激活   │ │激活   │ │待激活 │    │
│   └───────┘ └───────┘ └───────┘    │
└─────────────────────────────────────┘
```

### 用户激活 vs 账户激活

| 维度 | 用户激活 | 账户激活 |
|------|----------|----------|
| **主体** | 单个终端用户 | 整个公司/团队 |
| **标准** | 个人习惯建立 | 多用户 + 跨部门 |
| **价值** | 留存基础 | 扩展和变现基础 |
| **时间线** | 通常更短 | 通常更长 |

## 激活指标设计

### 好的激活指标特征

1. **可衡量** - 能在数据中清晰追踪
2. **可预测** - 与长期留存/变现强相关
3. **可行动** - 团队能够影响这个指标
4. **有时限** - 有明确的时间窗口

### 激活指标公式

```
激活 = 在 [时间窗口] 内
       完成 [核心价值行为]
       至少 [N次]
```

**示例**:
- Slack: 团队在前7天发送2000条消息
- Dropbox: 用户在第一天上传至少一个文件到至少一个设备

---

## 激活与增长漏斗

```
注册 → 入职 → Aha时刻 → 激活 → 留存 → 变现
           └──激活区间──┘
```

**关键洞察**: 激活是一个区间，不是一个点。从首次获得价值到建立习惯都属于激活。
