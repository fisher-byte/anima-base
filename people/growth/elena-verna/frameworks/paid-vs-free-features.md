# Paid vs Free Feature Decision Tree

**来源**: Elena Verna, "Should your new feature be free or paid?" (2024)

---

## 框架概述

在 Freemium 模式下，每个新功能都面临一个关键决策：应该免费提供还是付费？这个框架提供了一个一致的决策方法。

> "A consistent framework for deciding which of your new features to paywall."
> — Elena Verna

## 四个关键问题

### Q1: 功能是否支持核心激活？

**如果 YES → 免费**

- 用户需要这个功能才能体验核心价值
- 没有它用户无法完成激活
- 属于"Aha 时刻"的必要组成

**示例**: 
- Dropbox: 文件上传和同步
- Slack: 发送消息和创建频道

### Q2: 功能是否支持增长循环？

**如果 YES → 免费**

- 功能能够带来新用户
- 支持病毒传播或邀请
- 增加产品曝光

**示例**:
- Dropbox: 分享链接功能
- Calendly: 日历链接预约

### Q3: 功能是否满足企业级需求？

**如果 YES → 付费**

- 安全、合规、管理需求
- 大规模部署需要的功能
- 企业买家在意，终端用户不太在意

**示例**:
- SSO 单点登录
- 审计日志
- 管理控制台

### Q4: 功能是否属于高级/Power User？

**如果 YES → 付费**

- 超出基本使用场景
- 重度用户才需要
- 提供差异化价值

**示例**:
- 高级分析报告
- 自动化工作流
- API 访问

## 决策矩阵

| 功能特征 | 定价建议 | 原因 |
|----------|----------|------|
| 核心激活必需 | 免费 | 用户需要体验价值 |
| 支持增长循环 | 免费 | 帮助获取新用户 |
| 企业安全合规 | 付费 | 企业预算支付 |
| Power User 功能 | 付费 | 展示升级价值 |
| 存储/用量限制 | 按量付费 | 使用越多支付越多 |

## 常见错误

### 错误 1: 把激活必需功能付费
- 结果：用户无法体验价值
- 影响：激活率下降，留存恶化

### 错误 2: 把增长循环功能付费
- 结果：限制了产品的自然传播
- 影响：获客成本上升

### 错误 3: 把企业功能免费
- 结果：放弃了收入机会
- 影响：变现能力下降

### 错误 4: 功能太多付费
- 结果：免费版价值太低
- 影响：用户不愿试用

## 实施建议

### 新功能上线流程
1. **明确功能定位** - 它解决什么问题？
2. **走决策树** - 按顺序回答四个问题
3. **验证假设** - 用数据确认决策正确性
4. **迭代调整** - 根据数据调整定价

### 定期审计
- 每季度回顾现有功能定价
- 免费功能是否支持激活和增长？
- 付费功能是否真的值得付费？

---

## 边界案例处理

### 功能同时满足多个条件
- 优先保证激活和增长
- 可以通过用量限制实现差异化

### 竞品提供免费但你想收费
- 如果是激活/增长必需，跟随竞品免费
- 如果是企业/高级功能，可以收费

### 不确定时
- 倾向于先免费
- 观察数据后再决定是否收费
