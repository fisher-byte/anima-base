## 📚 一、决策证据库（42个案例）

### 分类1：战略决策证据（8个案例）

#### 1.1 防御性创新
**案例**：[张小龙-01] 视频号战略
**核心证据**：
- 抖音DAU 6亿威胁微信内容生态
- 决策：推出视频号防御（2020年上线）
- 代价：初期DAU低（<5000万）
- 结果：2年后DAU 5亿+，保住内容阵地

**可迁移性**：当核心业务受新兴产品威胁时的战略应对

#### 1.2 侧翼切入战略
**案例**：[张小龙-14] 微信支付
**核心证据**：
- 支付宝占据电商支付（优势明显）
- 决策：切入社交支付场景（转账/红包）
- 数据：春节红包一夜绑卡8000万张
- 结果：社交支付市场份额>支付宝

**可迁移性**：避开正面战场，寻找差异化场景

#### 1.3 平台去中心化治理
**案例**：[张小龙-21] 平台治理哲学
**核心证据**：
- 微信公众号：无编辑推荐（去中心化）
- 小程序：无应用商店（去中心化）
- 结果：生态多样化，头部集中度低

**可迁移性**：平台治理应选择"服务者"而非"管理者"角色



### 分类3：增长策略证据（7个案例）

#### 3.1 留存优先原则
**案例**：[Lenny-13] Elena Verna留存优先策略
**核心证据**：
- Miro实践：暂停50%获客预算 → 转投留存优化
- 识别Magic Moment：创建3个Board（留存率20% → 70%）
- Cohort分析：按注册时间分组监控留存
- 结果：留存翻倍 → 2年内ARR $50M→$200M（3倍增长）

**数据支撑**：
- 留存提升10% → 收入提升30%+
- 获客提升10% → 收入仅提升10%
- 复活流失用户成本 = 获取新用户成本的1/3-1/5

**可迁移性**：SaaS订阅模式通用原则

#### 3.2 增长渠道选择
**案例**：[Lenny-11] Casey Winters增长渠道
**核心证据**：
- 原则：一次只精通1个渠道（70-80%资源）
- 判断标准：CAC/LTV < 0.3才算成功
- 优势契合度 > 渠道热度（选你有优势的）
- Superhuman案例：推荐贡献80%+增长

**可迁移性**：适用资源有限的团队（<10人）

#### 3.3 上瘾模型（Hook循环）
**案例**：[Lenny-14] Nir Eyal习惯养成
**核心证据**：
- Hook循环：Trigger → Action → Variable Reward → Investment
- Instagram实践：
  * Trigger: Push通知 → 内部触发（无聊时自动打开）
  * Action: <1秒打开Feed
  * Variable Reward: 点赞数不确定（有时10个，有时1000个）
  * Investment: 发照片 → 等待点赞 → 下次Trigger
- 结果：用户每天自主打开10+次

**可迁移性**：高频产品习惯养成设计

#### 3.4 PLG策略
**案例**：[Lenny-18] Wes Bush Product-Led Growth
**核心证据**：
- TTV<5分钟是及格线（Time to Value）
  * Slack: 5分钟发第一条消息
  * Zoom: 1分钟开第一个会议
- Freemium vs Free Trial选择：
  * Freemium适用：边际成本低+网络效应
  * Free Trial适用：功能复杂需时间体验
- Viral Loop设计：Slack邀请同事 → 指数增长

**数据支撑**：Slack/Zoom从0到$1B ARR无销售团队

**可迁移性**：SaaS工具类产品



### 分类4：定价策略证据（1个案例）

#### 4.1 Value-Based Pricing
**案例**：[Lenny-16] Patrick Campbell定价策略
**核心证据**：
- 定价对收入影响：定价优化10% → 收入提升30%+（高于获客/留存）
- Value Metric选择：
  * Slack: 活跃用户数（而非总用户数）
  * AWS: 使用量
  * ProfitWell: 客户MRR规模
- 价格本地化数据：
  * 美国：$100/月（基准）
  * 印度：$30/月（30%，根据购买力）
  * 结果：收入提升20-30%
- 3档位定价最优：
  * 60-70%用户选中间档（Professional）
  * 避免2档（太少）或4+档（太多）
- 年付折扣：20%最优（2个月免费）
  * <15%吸引力不够
  * >25%损失过多
- 涨价策略：
  * Slack涨价20% → Churn仅增加3% → 收入增加17%

**可迁移性**：SaaS订阅制产品通用



### 分类5：产品设计证据（12个案例）

#### 5.1 克制设计原则
**案例**：[张小龙-18] 运营克制决策
**核心证据**：
- 不做红点提示（除非真实消息）
- 不做热点推送
- 朋友圈无"X条新动态"提示
- 限制公众号推送：订阅号每天1次，服务号每月4次
- 代价：短期DAU劣势（2014年微信4.4亿 vs 手机QQ 5.5亿）
- 收益：长期留存高（2020年月活12亿，卸载率<2%）

**可迁移性**：社交/工具类产品（不适用媒体类）

#### 5.2 功能生命周期管理
**案例**：[张小龙-19] 功能演进哲学
**核心证据**：
- 摇一摇生命周期：
  * 2011: 爆款（日活5000万）
  * 2015: 衰退（日活降至1000万，下降80%）
  * 2015 Q4: 降级（从一级入口 → 搜索框）
  * 2020: 持续保留但不推广（日活<500万）
- 漂流瓶下线：
  * 2019: 变成垃圾信息温床
  * 决策：直接下线（勇敢砍功能）
- 启发式：功能日活下降50%+ → 考虑降级或下线

**可迁移性**：成熟产品功能管理

#### 5.3 社交设计原则
**案例**：[张小龙-20] 社交设计原则
**核心证据**：
- 双向验证 vs 单向关注：
  * 微信：加好友需对方同意（真实关系）
  * 微博：单向关注（虚拟关系）
  * 结果：微信留存高，微博易疲劳
- 朋友圈设计：
  * 只有好友可见（不是公开广场）
  * 不能转发（避免病毒传播）
  * 不显示点赞总数（只显示谁点赞）
  * 可设置"三天可见"（30%+用户开启）
- 不做已读回执：避免社交压力

**可迁移性**：私密社交产品

### 场景12：设计评审

**证据来源**：1个案例
**决策工具**：
1. 3W准备 [Lenny-15]
   - What: 解决什么问题？
   - Where: 当前哪个阶段？

**关键原则**：
- 反馈必须达到Level 3+（actionable）
- 区分阶段（早期反馈方向，定稿反馈细节）

**适用条件**：

**适用条件**：

- 内容生态
- 成熟用户基础

## 🛠️ 四、决策工具箱（6大工具）

### 工具1: PMF验证工具
**来源**: [Lenny-01] Superhuman
**适用场景**: 早期产品验证PMF

**使用方法**:
1. 发PMF Survey给用户
2. 问："如果不能再使用这个产品，你会？"
   - Very disappointed
   - Somewhat disappointed
   - Not disappointed
3. 如果>40%选"Very disappointed"→达到PMF
4. 优化方向：聚焦"Somewhat disappointed"用户反馈

**典型数据**:
- Superhuman: 58%（达到PMF）
- 双边市场需分别测两侧

### 工具2: 优先级排序工具（LNO框架）
**来源**: [Lenny-02] Shreyas Doshi
**适用场景**: 个人/团队工作优先级

**使用方法**:
1. 列出所有工作项
2. 分类：
   - Leverage: 小投入大产出（优先做）
   - Neutral: 必要但无杠杆（适度做）
   - Overhead: 纯负担（减少或不做）
3. 时间分配：70% Leverage + 20% Neutral + 10% Overhead

**典型应用**:
- 个人工作规划（每周Review）
- 团队季度规划

### 工具3: Pre-mortem风险识别
**来源**: [Lenny-06] Shreyas Doshi
**适用场景**: 新项目启动前风险识别

**使用方法**:
1. 项目启动前，团队会议
2. 假设："6个月后项目失败了"
3. 每人独立写下失败原因（5-10分钟）
4. 集体分享，归类风险
5. 制定缓解方案

**关键技巧**:
- 独立书写（避免从众）
- 具体化风险（不是"可能失败"，而是"因为XX导致失败"）

### 工具4: 产品定位5步法
**来源**: [Lenny-10] April Dunford
**适用场景**: 产品定位不清晰时

**使用方法**:
1. Step 1: 列出独特属性（你有什么别人没有？）
2. Step 2: 转化为价值（这对客户意味着什么？）
3. Step 3: 识别竞争替代方案（客户会考虑什么替代？）
4. Step 4: 定义目标客户（谁最在乎你的价值？）
5. Step 5: 选择市场类别

**关键洞察**:
- 竞争替代方案 ≠ 行业竞品
- 示例：Slack竞品是"邮件"而非"Teams"

**来源**: [Lenny-08] Rahul Vohra

### 工具6: 增长渠道评估矩阵
**来源**: [Lenny-11] Casey Winters
**适用场景**: 选择主要增长渠道

**使用方法**:
1. 列出所有可能渠道
2. 评估2个维度：
   - 优势契合度（1-10分）: 你在这个渠道有多少优势？
   - CAC/LTV比率: 测试后计算
3. 选择得分最高的1个渠道
4. 70-80%资源投入主渠道
5. 测试标准：CAC/LTV < 0.3

**决策矩阵**:
```
渠道          优势契合度    CAC/LTV    决策
内容营销      9/10         0.2        ✅ 主渠道
SEO          7/10         0.35       ⚠️ 次要
付费广告      4/10         0.6        ❌ 放弃
```

## 📊 五、数据基准库（关键指标）

### PMF指标
- **40% PMF阈值**: >40%用户选"Very Disappointed"
- **NPS基准**: >50分算优秀（B2B SaaS）

### 留存指标
- **Day 7留存**: >40%（SaaS及格线）
- **Month 3留存**: >60%（健康产品）
- **Magic Moment效应**: 达到vs未达到，留存率差3-5倍

### 增长指标
- **CAC/LTV**: <0.3算成功，0.3-0.5可优化，>0.5砍掉
- **复活成本**: = 新用户获客成本的1/3-1/5
- **TTV基准**: <5分钟（PLG产品及格线）

### 定价指标
- **年付折扣**: 20%最优（2个月免费）
- **3档位转化**: 60-70%用户选中间档
- **涨价Churn**: 通常仅增加2-5%，但收入增加15-20%
- **价格本地化**: 收入提升20-30%

### 激活指标
- **Onboarding步骤**: 每增加1步 → 流失率+10%
- **最优步骤数**: <5步
- **Cohort-Based完课率**: 82% vs Self-Paced <10%

### 产品设计指标
- **微信三天可见**: 30%+用户开启
- **微信卸载率**: <2%（行业最低）
- **功能衰退信号**: 日活下降50%+ → 考虑降级
