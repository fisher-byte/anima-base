> **决策类型**: 用户激活优化

## 核心问题

**Onboarding的困境**:
```
问题：用户注册后流失
- 注册后流失率60-90% ❌
- 没达到Aha Moment
- 不理解产品价值

根因：Onboarding设计差
```

---

## Bowling Alley Framework（保龄球框架）

**类比**: Onboarding像保龄球

```
传统Onboarding（无护栏）:
- 用户像球，随机滚动
- 容易掉沟（流失）❌
- 很难击中目标（激活）

好的Onboarding（有护栏）:
- 设计护栏引导用户
- 减少掉沟风险 ✅
- 引导至Aha Moment

护栏 = 产品设计引导机制
```

### 4种护栏（Bumpers）

#### 1. Straight-Line Onboarding（直线引导）

**定义**: 强制用户完成关键步骤

```
案例 - Slack:
注册后强制：
1. 创建workspace
2. 邀请至少1个成员
3. 发送第一条消息

用户无法跳过 → 确保达到Aha Moment
```

#### 2. Welcome Mat（欢迎毯）

**定义**: 清晰展示产品价值

```
案例 - Notion:
首次打开：
1. 展示示例页面（而非空白）
2. 说明Notion能做什么
3. 提供模板（快速开始）

用户立即看到价值 → 降低认知成本
```

#### 3. Product Tours（产品导览）

**定义**: 交互式引导

```
❌ 差的Product Tour:
- 10步教程，用户跳过

✅ 好的Product Tour:
- 3步核心功能
- 边做边学（Learning by Doing）
- 可随时跳过
```

#### 4. Empty States（空状态设计）

**定义**: 无数据时的引导

```
❌ 差的Empty State:
- "您还没有数据"（无指引）

✅ 好的Empty State:
- "点击'+'创建第一个项目"
- 提供示例数据
- CTA按钮醒目
```

## 决策启发式

### 1. 识别Aha Moment

```
IF 优化激活
THEN 先识别Aha Moment：
  - 用户体验到核心价值的瞬间
  - 示例：Slack发第一条消息
  - 示例：Dropbox同步第一个文件

然后：引导用户最快达到Aha Moment
```

### 2. 减少Time to Value

```
IF 设计Onboarding
THEN 减少步骤至<5步：
  - 每增加1步 → 流失率+10%
  - 目标：3步内达到Aha Moment

案例：Zoom
1. 点击"新会议"
2. 分享链接
3. 开始会议 ✅ (TTV<1分钟)
```

### 3. 提供示例数据

```
IF 产品是空白画布
THEN 提供示例数据/模板：
  - Notion：示例页面
  - Figma：示例设计文件
  - Airtable：示例数据库

用户看到"填充状态" → 更容易理解价值
```

**案例总结**:
- 决策: 用Bowling Alley Framework优化激活
- 方法: 4种护栏（直线引导/欢迎毯/导览/空状态）
- 目标: 最快达到Aha Moment
- 启发式: TTV<5分钟，步骤<5步
- 适用: PLG产品激活优化
