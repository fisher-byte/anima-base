# Case 18: Wes Bush PLG策略 — Product-Led Growth的系统方法

> **决策者**: Wes Bush (《Product-Led Growth》作者)  
> **案例来源**: Lenny's Podcast + PLG框架  
> **决策类型**: 增长战略  
> **证据等级**: A级（Slack/Dropbox/Zoom案例）  
> **影响**: PLG成为SaaS主流增长模式

---

## 核心问题

**传统Sales-Led vs Product-Led**

```
Sales-Led Growth（传统）:
- 销售驱动
- 高客单价（>$10K/年）
- 销售周期长（3-6个月）
- 需要大量销售团队

Product-Led Growth（新模式）:
- 产品驱动
- 免费试用/Freemium
- 用户自助购买
- 产品即是销售
```

---

## PLG核心原则

### 1. 降低Time to Value（TTV）

**定义**: 用户从注册到获得价值的时间

```
IF 希望PLG成功
THEN 降低TTV至<5分钟：
  - Slack：5分钟内发第一条消息
  - Zoom：1分钟内开第一个会议
  - Figma：3分钟内创建第一个设计

对比：传统SaaS需要数周Onboarding
```

### 2. Freemium/Free Trial

**两种免费策略**:
```
Free Trial（限时免费）:
- 14/30天试用
- 时间到期后必须付费
- 适用：功能复杂，需要时间体验

Freemium（功能免费）:
- 基础功能永久免费
- 高级功能付费
- 适用：网络效应产品（Slack/Zoom）
```

### 3. Viral Loop（病毒循环）

**设计产品内传播机制**:
```
Slack病毒循环：
1. 用户A创建workspace
2. 邀请同事B/C/D加入
3. B/C/D体验后，在自己公司创建workspace
4. 继续邀请更多人...

结果：用户自增长，无需销售
```

---

## 决策启发式

### 1. TTV优先

```
IF 做PLG产品
THEN TTV<5分钟是及格线：
  - 简化注册（Email即可，无需表单）
  - 跳过Onboarding（直接进产品）
  - 快速达到Aha Moment
```

### 2. Freemium判断标准

```
IF 考虑Freemium
THEN 评估：
  - 边际成本低？（Cloud产品✅）
  - 有网络效应？（协作工具✅）
  - 用户量大？（TO C ✅）
  
满足2+个 → 适合Freemium
```

### 3. 产品内增长机制

```
IF 设计PLG产品
THEN 内置增长机制：
  - 邀请同事（Slack）
  - 分享链接（Notion/Figma）
  - 品牌曝光（Zoom会议底部logo）
```

---

**案例总结**:
- 决策: 用PLG替代Sales-Led增长
- 方法: 降低TTV + Freemium + Viral Loop
- 案例: Slack/Zoom从0到$1B ARR无销售团队
- 启发式: TTV<5分钟是PLG成功关键
- 适用: SaaS工具类产品（协作/设计/会议）
