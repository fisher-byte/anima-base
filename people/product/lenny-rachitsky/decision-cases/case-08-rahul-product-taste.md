## 1. 决策背景

### PM的困境
Rahul观察到产品经理的核心问题：
- **执行能力强**: 会写PRD、画原型、管项目
- **品味缺失**: 不知道什么是"好产品"vs"坏产品"
- **依赖他人判断**: 需要设计师/CEO告诉他"这个设计好不好"

### Rahul的洞察
> "Product taste is not innate. It's a skill you can develop through deliberate practice."  
> — Rahul Vohra

**核心问题**: 如何系统性培养产品品味？

## 2. 培养框架

### 方法1: The "10 Screenshots" Exercise

**操作步骤**:
1. **选定类别**: 如"登录页面"、"定价页"、"Onboarding流程"
2. **收集10个案例**: 5个你认为好的 + 5个你认为差的
3. **逐一分析**: 为什么这个好？为什么那个差？
4. **提炼原则**: 写下你的设计原则（如"登录页不应该有注册字段"）

**关键**: 不是随便看，而是**对比+分析+提炼**

**Rahul的实践**:
- 每周做一次"10 Screenshots"练习
- 持续2年，建立了个人设计原则库
- Superhuman的设计就是这些原则的应用

### 方法2: "Build Your Swipe File"（灵感文件）

**操作**:
- 看到好的设计/文案/交互 → 立即截图保存
- 分类整理（Onboarding/定价/空状态/错误提示等）
- 需要设计时，先看Swipe File找灵感

**工具推荐**:
- Notion数据库
- Pinterest Board
- Apple Notes（最简单）

### 方法3: "Taste Stack"（品味分层）

**品味分为3层**:
1. **Surface Taste（表层）**: 知道"这个好看"vs"那个丑"（大多数人停留在这）
2. **Structural Taste（结构）**: 知道"为什么好看"（布局/色彩/留白）
3. **Strategic Taste（战略）**: 知道"为什么这样设计"（符合用户心智/业务目标）

**培养路径**: 必须从Layer 1→2→3逐步深入，不能跳级

---

## 3. 实际案例

### Superhuman的邮件设计

**问题**: 如何设计邮件列表？  
**Rahul的流程**:
1. **收集对比**: 研究Gmail/Outlook/Apple Mail/Spark等10+邮件客户端
2. **分析差异**: 为什么Gmail用对话模式？为什么Outlook用单封邮件模式？
3. **提炼原则**: "邮件列表应该是扫描性的，不是阅读性的"
4. **设计应用**: Superhuman采用单行显示、快捷键导航

**结果**: Superhuman的邮件体验被认为是行业最佳

## 4. 决策启发式

### 启发式1: 对比学习法（不是盲目模仿）

**适用场景**: 学习任何产品设计  
**决策规则**:
```
IF 想提升产品品味
THEN 对比学习：
  1. 收集10个同类产品
  2. 5个好 + 5个差（强制分类）
  3. 逐一分析：为什么好？为什么差？
  4. 提炼原则：写下你的设计原则
```

**关键**:
- ❌ 看1000个产品但不思考 → 无用
- ✅ 对比10个产品并深度分析 → 建立品味

### 启发式2: 建立个人"设计原则库"

**决策规则**:
```
IF 分析了足够多案例
THEN 将洞察固化为原则：
  - "登录页不应该超过3个字段"
  - "Onboarding不应该超过3步"
  - "定价页必须有对比（而非单一选项）"
```

**Rahul的原则库**（部分）:
- Email设计: "一行一封邮件，不展开内容"
- Onboarding: "0状态必须展示终态样子（不是空白）"
- 错误提示: "告诉用户怎么做，不是告诉他做错了"

### 启发式3: 从表层→结构→战略（分层培养）

**决策规则**:
```
IF 评价一个设计
THEN 分3层分析：
  - Layer 1 (表层): 这个好看吗？→ 训练直觉
  - Layer 2 (结构): 为什么好看？→ 训练分析力
  - Layer 3 (战略): 为什么这样设计？→ 训练战略思维
```

**不要跳级**: 很多PM想直接做战略判断，但缺乏表层和结构判断能力，导致判断失准。
