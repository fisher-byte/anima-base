## 核心框架

### GEM模型（产品战略）

**定义**: Get/Engagement/Monetization

```
产品战略3要素：
1. Get（获客）- 如何获取用户？
2. Engagement（参与）- 如何提升留存？
3. Monetization（变现）- 如何赚钱？

关键：3个要素要平衡，不能只优化一个
```

**Netflix案例**:
```
Get: 
- 免费试用30天
- 朋友推荐

Engagement:
- 个性化推荐
- 自动播放下一集
- 优质原创内容

Monetization:
- 订阅模式（$8-20/月）
- 不同档位定价
```

### DHM模型（功能决策）

**定义**: Delight/Hard-to-Copy/Margin-Enhancing

```
IF 评估新功能
THEN 问3个问题：
1. Delight（愉悦）- 用户喜欢吗？
2. Hard-to-Copy（护城河）- 竞品容易抄吗？
3. Margin-Enhancing（利润）- 提升利润吗？

理想功能：3个都满足 ✅
```

**Netflix个性化推荐评估**:
```
1. Delight: ✅ 用户找片更快，满意度提升
2. Hard-to-Copy: ✅ 需要大量数据和算法（护城河）
3. Margin-Enhancing: ✅ 提升留存，降低Churn

结论：值得大量投资
```

---

## 决策启发式

### 1. GEM平衡

```
IF 设计产品战略
THEN 平衡GEM 3要素：
  - 不能只优化获客（忽略留存）
  - 不能只优化留存（忽略变现）
  - 需要3者协同
```

### 2. DHM评估功能

```
IF 评估新功能优先级
THEN 用DHM打分：
  - 3个都满足：最高优先级 ✅
  - 满足2个：中等优先级 ⚠️
  - 只满足1个：低优先级或不做 ❌
```

### 3. 数据告诉你"What"，洞察告诉你"Why"

```
IF 看到数据异常
THEN 不要只看数字，要理解原因：
  - 数据：用户Churn率上升10%（What）
  - 洞察：因为竞品推出新功能（Why）
  - 行动：快速跟进（How）
```
