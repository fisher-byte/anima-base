## 概述

North Star Metric (NSM) 框架是 Lenny Rachitsky 基于对 40+ 成功增长期公司调研总结的指标选择方法论。这个框架帮助团队选择一个核心指标，指引所有产品和增长决策。

**核心理念：** "Your North Star Metric is your strategy, and your strategy is your North Star Metric."（你的北极星指标就是你的战略，你的战略就是你的北极星指标）

## 什么是 North Star Metric

### 定义
North Star Metric 是一个单一的关键指标，最能代表：
- 你为用户创造的核心价值
- 公司的长期增长
- 团队的共同目标

### 好的 NSM 特征

1. **反映核心价值** - 用户体验到的主要价值
2. **预示长期成功** - 与收入和增长高度相关
3. **可操作** - 团队能影响这个指标
4. **易于理解** - 全公司都能理解
5. **可测量** - 能准确追踪

## 如何选择 North Star Metric

### 步骤 1：理解你的产品价值

使用 **Jobs to be Done (JTBD)** 框架：
- 用户"雇佣"你的产品完成什么任务？
- 什么时刻用户感受到了价值？（Aha Moment）
- 用户持续回来的原因是什么？

### 步骤 2：映射用户旅程

识别关键里程碑：
1. 注册/安装
2. 首次体验核心价值
3. 建立习惯
4. 持续活跃
5. 深度使用

### 步骤 3：选择代表核心价值的指标

**原则：**
- 专注于"活跃"而非"注册"
- 专注于"价值交换"而非"停留时间"
- 专注于"深度使用"而非"浅层互动"

### 步骤 4：验证指标

问自己：
- 如果这个指标增长，业务一定会增长吗？
- 团队能影响这个指标吗？
- 这个指标能长期使用吗（不需要频繁更换）？

## 常见行业的 North Star Metric

基于 Lenny 对 40+ 公司的调研：

### 1. 消费社交类
- **Airbnb**: Nights Booked（预订晚数）
- **WhatsApp**: Number of Sends（发送消息数）
- **Facebook**: Daily Active Users（日活用户）
- **Instagram**: Daily Active Users（日活用户）
- **TikTok**: Time Spent（使用时长）

### 2. SaaS/生产力工具
- **Slack**: Messages Sent（发送消息数）
- **Notion**: Weekly Active Users（周活用户）
- **Figma**: Files Created（创建文件数）
- **Zoom**: Weekly Meeting Participants（周会议参与人数）
- **Dropbox**: Active Users（活跃用户）

### 3. 电商/交易平台
- **Amazon**: Number of Purchases（购买次数）
- **Uber**: Rides per Week（每周订单数）
- **DoorDash**: Orders per Month（每月订单数）
- **Etsy**: Gross Merchandise Sales (GMS)（商品交易总额）

### 4. 内容/媒体
- **Spotify**: Time Spent Listening（收听时长）
- **Netflix**: Hours Streamed（观看时长）
- **Medium**: Total Time Reading（阅读总时长）
- **YouTube**: Watch Time（观看时长）

### 5. B2B SaaS
- **HubSpot**: Weekly Active Teams（周活跃团队）
- **Asana**: Tasks Completed（完成任务数）
- **Salesforce**: Opportunities Created（创建商机数）

## 指标类型分类

### Type 1: 使用频率类
**适用：** 高频使用的产品
- Daily Active Users (DAU)
- Weekly Active Users (WAU)
- Monthly Active Users (MAU)

**优点：** 简单直接
**缺点：** 可能不反映使用深度

### Type 2: 核心动作类
**适用：** 有明确价值动作的产品
- Messages Sent
- Nights Booked
- Rides Completed
- Files Created

**优点：** 直接反映价值创造
**缺点：** 可能忽视用户质量

### Type 3: 消费类
**适用：** 消费型内容产品
- Time Spent
- Content Consumed
- Sessions per User

**优点：** 反映用户参与度
**缺点：** 不一定等于价值

### Type 4: 增长效率类
**适用：** 增长期公司
- L7 / L28（7天活跃 / 28天活跃）
- Percent of Users Active Each Week
- Retained Revenue

**优点：** 综合反映健康度
**缺点：** 较难理解

## 常见陷阱

### 1. 选择虚荣指标
❌ 总注册用户数
❌ 总下载次数
❌ 页面浏览量（PV）

✅ 活跃用户数
✅ 核心价值获取次数
✅ 留存率相关指标

### 2. 指标过于复杂
❌ 需要复杂公式计算
❌ 包含多个变量
❌ 团队理解困难

✅ 一句话能说清楚
✅ 所有团队都能理解
✅ 易于追踪和报告

### 3. 指标与商业模式脱节
❌ NSM 增长但收入不增长
❌ NSM 增长但用户质量下降
❌ NSM 增长但成本不可控

✅ NSM 与 LTV 高度相关
✅ NSM 增长带动收入增长
✅ NSM 可持续增长

### 4. 过于频繁更换
❌ 每个季度换一次
❌ 遇到困难就换指标

✅ 至少使用 1-2 年
✅ 只在战略调整时改变

## 多个 North Star Metric？

### Lenny 的建议：
**理想情况：1 个 NSM**
- 最简单
- 最易对齐
- 最易执行

**可接受：2-3 个 NSM**
- 适用于多产品公司
- 适用于多边平台（如 Airbnb：房东 + 房客）
- 需要清晰的优先级

**不推荐：4+ 个 NSM**
- 失去焦点
- 团队困惑
- 等于没有 NSM

### 多 NSM 场景

**双边市场例子：Airbnb**
- 主 NSM: Nights Booked
- 次级指标: 
  - Supply side: Available Nights
  - Demand side: Searches

**多产品例子：Facebook**
- Facebook: DAU
- Instagram: DAU
- WhatsApp: Messages Sent
- 集团层面: Family DAU

## 与 OKR 的关系

### NSM vs OKR
- **NSM** = "What"（做什么）- 长期不变的核心指标
- **OKR** = "How"（怎么做）- 季度性的具体目标和关键结果

### 协同工作
```
North Star Metric: Weekly Active Users

Q1 OKR:
Objective: 提升新用户留存
Key Results:
- D7 留存从 40% 提升到 50%
- D30 留存从 25% 提升到 35%
- 新用户 WAU 增长 30%

Q2 OKR:
Objective: 提升老用户回访
Key Results:
- Reactivation rate 提升 20%
- L7/L28 从 0.3 提升到 0.4
- 老用户 WAU 增长 25%
```

## 实施建议

### 1. 全公司对齐
- CEO 清晰传达 NSM
- 定期在全员会议上回顾
- 各团队 OKR 都连接到 NSM

### 2. 可视化展示
- 办公室大屏实时显示
- Dashboard 首页展示
- Slack bot 定期播报

### 3. 分解到团队
每个团队理解自己如何影响 NSM：
- 产品团队 → 通过功能创新
- 增长团队 → 通过渠道优化
- 工程团队 → 通过性能提升
- 设计团队 → 通过体验优化

### 4. 定期回顾
- 每周查看趋势
- 每月深度分析
- 每季度战略复盘

## 案例分析

### Airbnb: Nights Booked

**为什么选择这个指标：**
- 直接反映核心价值（住宿预订）
- 同时衡量供需两侧（房东 + 房客）
- 与收入直接相关
- 可长期使用

**如何影响 NSM：**
- 增加高质量房源（供给侧）
- 提升搜索和发现（需求侧）
- 优化预订转化
- 改善用户体验促进复购

### Slack: Messages Sent

**为什么选择这个指标：**
- 直接反映团队协作价值
- 高频可测量
- 与付费转化高度相关
- 易于理解

**如何影响 NSM：**
- 提升 Onboarding 质量
- 增加集成数量
- 优化通知策略
- 建立使用习惯

## 进阶思考

### 1. NSM 的演化
随着公司发展，NSM 可能需要调整：
- **Early stage**: 专注用户获取和激活
- **Growth stage**: 专注活跃和留存
- **Mature stage**: 专注变现和效率

### 2. NSM 的平衡
避免过度优化单一指标：
- 设置护栏指标（Guardrail Metrics）
- 关注用户质量
- 监控负面影响

### 3. NSM 与产品战略
NSM 是战略的体现：
- 改变 NSM = 改变战略
- 战略调整 → NSM 可能需要调整
- NSM 指引产品路线图

## 检查清单

选择 NSM 时问自己：

- [ ] 这个指标反映我们为用户创造的核心价值吗？
- [ ] 如果这个指标增长 2x，业务会显著增长吗？
- [ ] 全公司上下都能理解这个指标吗？
- [ ] 各团队都能找到影响这个指标的方法吗？
- [ ] 这个指标能使用至少 1-2 年吗？
- [ ] 这个指标易于测量和追踪吗？
- [ ] 这个指标与长期收入高度相关吗？
- [ ] 优化这个指标不会伤害用户体验吗？
