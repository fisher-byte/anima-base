## 🎯 核心场景启发式

### 场景1: PMF验证

#### 1.1 "40% Very Disappointed"启发式 [Case-01]
```
IF 做PMF Survey
AND >40%用户选"Very Disappointed"
THEN 达到PMF
```
- **来源**: Rahul Vohra (Superhuman)
- **证据等级**: A级
- **适用**: B2B/B2C产品
- **不适用**: 双边市场（需单独测两侧）

#### 1.2 "Somewhat Disappointed优先"启发式 [Case-01]
```
IF PMF后优化产品
THEN 聚焦"有些失望"用户（不是"很失望"或"不失望"）
```
- **理由**: "很失望"用户反馈会误导，"不失望"用户是lost cause
- **适用**: 产品迭代roadmap

#### 1.3 Solution Deepening First [Case-01]
```
IF 早期产品（< PMF）
THEN 先把产品做深 → 再拓展市场
```
- **案例**: Superhuman前几年只支持Gmail Web版
- **不适用**: 窗口期极短的市场

### 场景2: 产品定位

#### 2.1 竞争替代方案 > 行业竞品 [Case-10]
```
IF 定位产品
THEN 先问"客户在选我之前会考虑什么"
  ❌ 不是行业竞品列表
  ✅ 是真实替代方案
```
- **来源**: April Dunford
- **案例**: Slack竞品是"邮件"而非"Teams"

#### 2.2 收窄目标客户 > 服务所有人 [Case-10]
```
IF 定位目标客户
THEN 找最在乎你独特价值的人
  ❌ "服务所有邮件用户"
  ✅ "服务每天100+邮件的高管"
```

#### 2.3 定位顺序：独特属性→价值→客户 [Case-10]
```
正确顺序：
1. 独特属性（你有什么别人没有？）
2. 价值（这对客户意味着什么？）
3. 目标客户（谁最在乎这个价值？）
```
- **为什么**: 先定义客户容易陷入"服务所有人"陷阱

### 场景3: 增长渠道选择

#### 3.1 一次只精通一个渠道 [Case-11]
```
IF 资源有限（<10人团队）
THEN 只投入1个主渠道：
  - 70-80%资源（成为世界级）
  - 20-30%资源实验新渠道
  - 0%其他渠道
```
- **来源**: Lenny + Casey Winters
- **反例**: 5个渠道各20%资源 → 每个都平庸

#### 3.2 优势契合度 > 渠道热度 [Case-11]
```
IF 选择增长渠道
THEN 优先选择"你有优势"的渠道
  ❌ "抖音很火，我们也做抖音"
  ✅ "我们有10万Newsletter，加倍投入内容"
```

#### 3.3 CAC/LTV < 0.3 才算成功 [Case-11]
```
测试渠道判断标准：
- < 0.3: Great（加倍投入）
- 0.3-0.5: Promising（优化后决定）
- > 0.5: Weak（砍掉或转向）
```

### 场景4: 优先级排序

#### 4.1 Leverage/Neutral/Overhead分类 [Case-02]
```
IF 评估工作优先级
THEN 分类：
  - Leverage: 小投入大产出（优先）
  - Neutral: 必要但无杠杆（适度）
  - Overhead: 纯负担（减少）
```
- **来源**: Shreyas Doshi
- **适用**: 个人/团队优先级

#### 4.2 产品工作三层级 [Case-07]
```
PM工作分层：
- Level 1: Execution（执行）
- Level 2: Ownership（负责）
- Level 3: Vision（愿景）

晋升关键：花更多时间在更高层级
```
- **时间分配诊断**: 每周回顾L1/L2/L3时间占比

### 场景5: 风险识别

#### 5.1 Pre-mortem（前瞻性事后诸葛亮）[Case-06]
```
IF 即将启动新项目
THEN 项目启动前执行Pre-mortem：
  1. 假设"6个月后项目失败了"
  2. 团队独立写下失败原因
  3. 归类风险，制定缓解方案
```
- **来源**: Shreyas Doshi
- **为什么有效**: 打破群体思维，具体化风险

#### 5.2 独立书写 → 集体分享 [Case-06]
```
IF 需要团队识别风险
THEN 先独立、后集体
  ❌ 不要直接开会讨论风险（容易从众）
  ✅ 先让每人独立写，再一起分享
```

### 场景6: 决策质量

#### 6.1 决策质量 vs 结果质量解耦 [Case-05]
```
IF 决策有了结果
THEN 分开评估：
  - 决策质量: 给定当时信息，决策好坏？
  - 结果质量: 实际发生了什么？
  - 学习: 好决策+坏结果 = 继续该流程
```
- **来源**: Annie Duke
- **四象限**: 好决策×好结果 / 好决策×坏结果 / ...

#### 6.2 Edge-Downside-Upside 3问 [Case-05]
```
IF 考虑新机会
THEN 依次回答：
  1. Edge: 我的优势是什么？（无则放弃）
  2. Downside: 最坏情况能承受吗？（不能则放弃）
  3. Upside: 期望值为正吗？（否则放弃）
```

### 场景7: 战略决策

#### 7.1 避开正面战场，侧翼切入 [Case-14]
```
IF 挑战行业巨头
THEN 找差异化场景：
  - 支付宝: 电商支付
  - 微信: 社交支付（转账/红包）
```
- **来源**: 张小龙（微信支付）
- **关键**: 找对手"够不到"的场景

#### 7.2 用"特洛伊木马"突破防线 [Case-15]
```
IF 新业务用户接受度低
THEN 用趣味化包装：
  - 直接推绑卡 ❌ → 用户抗拒
  - 用红包包装 ✅ → 用户主动绑卡
```
- **案例**: 微信红包激活支付

#### 7.3 战略耐心 + 关键爆破点 [Case-14]
```
IF 新业务初期数据不佳
AND 战略方向正确
THEN 保持耐心，寻找爆破点
```
- **案例**: 微信支付忍受5个月低绑卡率，春节红包一举突破

### 场景8: 产品设计

#### 8.1 克制设计（功能限制）[Case-11/13/15/16/17]
```
IF 新功能可能被过度使用
AND 过度使用伤害体验
THEN 设置合理限制：
  - 语音60秒
  - 朋友圈9张图
  - 广告每天1-2条
```
- **来源**: 张小龙（多案例）
- **适用**: 社交/内容产品

#### 8.2 快速跟进 + 极致打磨 [Case-11]
```
IF 竞品已验证的创新功能
THEN 快速跟进（不要重新发明轮子）
BUT 在执行细节上做到极致
```
- **案例**: 微信语音消息跟进Talkbox

#### 8.3 克制是长期主义 [Case-08]
```
IF 可以通过激进功能快速增长
BUT 会损害长期用户体验
THEN 选择克制：
  - 看一看藏在"发现"页（非主入口）
  - 不主动推送（用户主动点击）
```
- **短期代价**: DAU低于竞品
- **长期收益**: 保持用户信任

### 场景9: 冷启动

#### 9.1 冷启动双轨策略 [Case-07]
```
IF 有存量资源（如QQ关系链）
AND 希望建立新生态
THEN 双轨策略：
  - 主路径: 新生态原生增长（通讯录）
  - 辅助路径: 存量资源导入（QQ好友）
  - 关键: 辅助路径是"可选"而非"默认"
```
- **来源**: 张小龙（微信冷启动）

#### 9.2 真实关系 > 平台关系 [Case-07]
```
IF 构建社交产品
THEN 优先真实关系：
  - 手机通讯录 > QQ好友
  - 线下认识 > 线上添加
  - 双向确认 > 单向关注
```

#### 9.3 渐进式去依赖 [Case-07]
```
新产品依赖母体资源冷启动流程：
- Phase 1 (0-6个月): 大量导入存量资源
- Phase 2 (6-18个月): 建立独立增长飞轮
- Phase 3 (18个月+): 降低导入入口优先级
```

### 场景10: 产品演进

#### 10.1 渐进式改革 vs 激进重构 [Case-06]
```
IF 需要重构核心功能
AND 用户有强烈使用习惯
THEN 渐进式改革：
  - Phase 1: 保留旧形态，增加新选项
  - Phase 2: 观察数据，逐步引导
  - Phase 3: 如果新形态成功，再替换旧形态
```
- **案例**: 订阅号改版（保留订阅号，独立做"看一看"）

#### 10.2 社交推荐 vs 算法推荐 [Case-06/08]
```
IF 需要做内容推荐
THEN 根据产品定位选择：
  - 纯算法推荐（今日头条）: 最大化消费
  - 社交推荐（微信）: 基于好友信任
  - 混合模式: 社交为主+算法辅助
```

#### 10.3 独立入口 vs 强制改版 [Case-06/08]
```
IF 新功能可能引发争议
THEN 做独立入口（而非强制替换）：
  - 给用户选择权
  - 降低反感度
  - 用数据验证价值
```

### 场景11: 能力培养

#### 11.1 产品品味对比学习法 [Case-08]
```
IF 想提升产品品味
THEN 对比学习：
  1. 收集10个同类产品（5好+5差）
  2. 逐一分析：为什么好？为什么差？
  3. 提炼原则：写下设计原则
```
- **来源**: Rahul Vohra
- **关键**: 对比+分析+提炼（不是盲目看）

#### 11.2 建立个人设计原则库 [Case-08]
```
IF 分析了足够多案例
THEN 将洞察固化为原则：
  - "登录页不应超过3个字段"
  - "Onboarding不应超过3步"
```

#### 11.3 品味三层级（表层→结构→战略）[Case-08]
```
IF 评价设计
THEN 分3层分析：
  - Layer 1: 这个好看吗？（训练直觉）
  - Layer 2: 为什么好看？（训练分析力）
  - Layer 3: 为什么这样设计？（训练战略思维）
```

### 场景12: 职业发展

#### 12.1 先验证后投入（不是All-in）[Case-09]
```
IF 新方向不确定性高
THEN 分阶段投入：
  - Phase 1: 业余时间验证（0-3个月）
  - Phase 2: 如果有信号，增加投入（3-6个月）
  - Phase 3: 如果达到阈值，全职投入（6个月+）
```
- **来源**: Lenny (Newsletter启动)
- **阈值示例**: 打开率>50%, 月收入>$15000

#### 12.2 选择Leverage最高的方向 [Case-09]
```
IF 评估多个职业方向
THEN 对比Leverage（影响力/投入时间）：
  - PM工作: 1个产品（Leverage中等）
  - Newsletter: 1篇文章影响数万人（Leverage高）
  - 咨询: 1对1服务（Leverage低）
```

#### 12.3 看"留存"而非"增长"（质量>数量）[Case-09]
```
IF 评估内容是否有价值
THEN 看留存指标：
  ❌ "我有1万订阅"（虚荣指标）
  ✅ "打开率>50%"（真实价值）
```

## 🔀 决策流程组合

### 组合1: PMF → 定位 → 增长

**完整流程**:
1. **验证PMF** [Case-01]: 用40% Survey确认达到PMF
2. **明确定位** [Case-10]: 用April 5步法定位产品
3. **选择渠道** [Case-11]: 找优势契合度最高的渠道
4. **成为世界级**: 70-80%资源投入主渠道

**案例**: Superhuman
- PMF: 40% Survey验证达到
- 定位: "最快的邮件客户端"（目标=高邮件量高管）
- 渠道: 推荐机制（NPS驱动）
- 结果: 推荐贡献80%+增长

### 组合2: 战略决策 → 风险识别 → 执行验证

**完整流程**:
1. **战略决策** [Case-01/02/03]: 确定产品方向
2. **Pre-mortem** [Case-06]: 识别潜在风险
3. **分阶段验证** [Case-09]: MVP测试 → 优化 → 规模化

**案例**: 微信视频号
- 战略: 防御抖音，保住内容阵地
- 风险识别: 用户体验冲突、内容供给不足、竞争压力
- 执行: 低调上线 → 逐步优化 → 独立Tab

### 组合3: 优先级排序 → 能力培养 → 职业发展

**完整流程**:
1. **评估当前工作** [Case-02]: 用LNO分类
2. **识别工作层级** [Case-07]: Execution/Ownership/Vision
3. **培养核心能力** [Case-08]: 产品品味/决策能力
4. **规划职业路径** [Case-09]: 选择Leverage最高的方向

**PM成长路径**:
- Junior: 优先Leverage工作 + 做好Execution
- Mid: 提升产品品味 + 承担Ownership
- Senior: 刻意做Vision工作 + 考虑职业转型
