## 🔑 核心决策启发式（Top 10）

### 1. 40% PMF阈值（Rahul Vohra）
```
IF 做PMF Survey
AND >40%用户选"Very Disappointed"
THEN 达到PMF
```
**适用**: B2B/B2C产品验证

### 2. Leverage/Neutral/Overhead分类（Shreyas Doshi）
```
IF 评估工作优先级
THEN 分类：
  - Leverage: 小投入大产出（优先做）
  - Neutral: 必要但不产生杠杆（适度做）
  - Overhead: 纯负担（尽量减少）
```
**适用**: 个人效能/团队优先级

### 3. 决策质量 vs 结果质量解耦（Annie Duke）
```
IF 决策有了结果
THEN 分开评估：
  - 决策质量: 给定当时信息，决策是好是坏？
  - 结果质量: 实际发生了什么？
  - 学习: 好决策+坏结果 = 继续该流程
```
**适用**: 任何决策复盘

### 4. Pre-mortem（Shreyas Doshi）
```
IF 即将启动新项目
THEN 时间旅行：
  1. 假设"6个月后项目失败了"
  2. 团队独立写下失败原因
  3. 归类风险，制定缓解方案
```
**适用**: 任何>2周的项目

### 5. 克制设计（张小龙，多案例）
```
IF 新功能可能被过度使用
AND 过度使用会伤害用户体验
THEN 设置合理限制：
  - 语音60秒
  - 朋友圈9张图
  - 广告每天1-2条
```
**适用**: 社交/内容产品

### 6. 战术性功能vs战略性产品（张小龙-附近的人）
```
IF 增长需求 + 战略冲突并存
THEN 用战术性功能满足短期需求：
  - 定位为"工具"（非核心）
  - 克制设计
  - 预留降权空间
```
**适用**: 需要短期增长但与长期战略冲突

### 7. 快速跟进 + 极致打磨（张小龙-语音）
```
IF 竞品已验证的创新功能
THEN 快速跟进（不要重新发明轮子）
BUT 在执行细节上做到极致
```
**适用**: 竞品已验证的功能

### 8. 先验证后投入（Lenny-Newsletter）
```
IF 新方向不确定性高
THEN 分阶段投入：
  - Phase 1: 业余时间验证（0-3个月）
  - Phase 2: 有信号则增加投入（3-6个月）
  - Phase 3: 达到阈值则全职（6个月后）
```
**适用**: 职业转型/创业方向选择

### 9. 产品品味对比学习法（Rahul-Superhuman）
```
IF 想提升产品品味
THEN 对比学习：
  1. 收集10个同类产品（5好+5差）
  2. 逐一分析：为什么好？为什么差？
  3. 提炼原则：写下设计原则
```
**适用**: PM能力提升

### 10. Edge-Downside-Upside 3问（Annie Duke）
```
IF 考虑新机会
THEN 依次回答：
  1. Edge: 我的优势是什么？（无则放弃）
  2. Downside: 最坏情况能承受吗？（不能则放弃）
  3. Upside: 期望值为正吗？（否则放弃）
```
**适用**: 投资/投入机会评估

1. **元信息**: 决策者/时间/类型/证据等级

4. **决策结果**: 短期/长期影响
