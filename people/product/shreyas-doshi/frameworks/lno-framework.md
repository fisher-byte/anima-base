# LNO Framework (Leverage, Neutral, Overhead)

## 框架概述

LNO Framework 是 Shreyas Doshi 创造的时间管理框架，帮助产品经理和领导者区分不同类型的任务并差异化投入精力。这个框架彻底改变了很多人对时间管理的认知。

## 核心理念

> "Not all tasks are created equal. The key is knowing which tasks deserve your best effort and which just need to get done."

传统时间管理的问题：
- 把所有任务当作同等重要
- 追求每件事都做到完美
- 精力分配不合理
- 导致疲惫和效率低下

## LNO 分类

### L - Leverage Tasks (高杠杆任务) 🔥

**定义**: 高回报任务，值得超额投入，做得好能产生巨大影响

**特征**:
- 结果有长期影响
- 质量差距明显
- 投入额外时间有高回报
- 通常涉及重要决策或创造性工作

**示例**:
- 制定产品战略
- 重要的招聘决策
- 关键客户演示
- 年度规划
- 处理重大危机

**投入策略**: 
- 投入**120%**的精力
- 追求卓越
- 不惜额外时间
- 寻求多方反馈

### N - Neutral Tasks (中性任务) ⚖️

**定义**: 需要做好但不需要做到极致的任务

**特征**:
- 做到"足够好"即可
- 额外投入回报递减
- 是维持运营的必要工作
- 可以标准化处理

**示例**:
- 常规状态更新
- 标准文档撰写
- 普通会议准备
- 日常决策
- 例行汇报

**投入策略**: 
- 投入**100%**的精力
- 达到标准即可
- 使用模板和流程
- 不追求完美

### O - Overhead Tasks (开销任务) 📋

**定义**: 必须完成但价值有限的任务

**特征**:
- 行政性工作
- 完成即可，质量影响小
- 占用时间但不产生直接价值
- 可以委托或自动化

**示例**:
- 费用报销
- 日程安排
- 常规邮件回复
- 表格填写
- 系统更新

**投入策略**: 
- 投入**80%**或更少
- 快速完成
- 能委托就委托
- 能自动化就自动化
- 批量处理

## 应用方法

### Step 1: 任务盘点
- 列出本周所有任务
- 不遗漏任何事项

### Step 2: LNO 分类
对每个任务问：
- 做得特别好会有显著额外回报吗？→ L
- 做到标准就够了吗？→ N
- 只要完成就行了吗？→ O

### Step 3: 差异化投入
- L任务：主动投入更多
- N任务：保持标准
- O任务：最小化时间

### Step 4: 日程安排
- 把L任务放在精力最好的时间
- N任务可以在一般时间
- O任务批量处理或委托

## 实际应用示例

### 产品经理的一周

```
L (Leverage) - 20% 时间，60% 精力
├── 产品战略评审
├── 重要客户访谈
└── 关键决策会议

N (Neutral) - 50% 时间，35% 精力
├── Sprint计划会
├── 日常stand-up
├── 需求文档
└── 利益相关者同步

O (Overhead) - 30% 时间，5% 精力
├── 邮件处理
├── 日程管理
├── 系统更新
└── 行政事务
```

## 常见误区

### ❌ 所有任务都当L处理
结果：精力耗尽，真正重要的事反而没做好

### ❌ 把L任务当N处理
结果：错失重大机会，影响长期发展

### ❌ 花太多时间在O上
结果：被琐事淹没，没时间做重要的事

### ❌ 分类太极端
正确做法：根据具体情况灵活调整

## 进阶技巧

### 1. 动态调整
同一类型的任务可能因情况变化而改变分类：
- 普通周报是N，但给新CEO的是L
- 日常邮件是O，但重要客户的是N

### 2. 上下级对齐
- 与上级讨论什么是L任务
- 确保对"足够好"有共识
- 在O任务上获得授权简化

### 3. 团队应用
- 分享框架给团队
- 帮助团队成员识别他们的L任务
- 减少团队在O任务上的投入
