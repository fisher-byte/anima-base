# Default Alive or Default Dead?

> 原文链接: [paulgraham.com/aord.html](https://paulgraham.com/aord.html)

## 概述

Paul Graham 提出了评估早期创业公司健康状况的最重要问题：**假设支出不变，按照过去几个月的收入增长速度，你能在现有资金耗尽之前实现盈利吗？** 换句话说——默认情况下，你的公司是活着还是死了？

## 核心概念

### Default Alive vs Default Dead

| 状态 | 定义 | 含义 |
|------|------|------|
| **Default Alive** | 按当前增长轨迹，能在钱花完前盈利 | 融资是选择，不是必须 |
| **Default Dead** | 按当前轨迹，钱会先花完 | 必须融资或改变，否则死亡 |

### 为什么这个问题如此重要

**令人震惊的发现**：
- 与创业公司交谈时，Graham 发现约一半的创始人不知道自己属于哪种状态
- 这是生死攸关的问题，却被忽视

**计算工具**：
- Trevor Blackwell 制作了一个[计算器](http://growth.tlb.org/)
- 输入：当前银行余额、月支出、月收入、增长率
- 输出：你是 default alive 还是 default dead

## 核心洞察

### 1. Default Dead 的危险

**融资不是确定的**：
- 如果你是 default dead，你的命运掌握在投资者手中
- 投资者可能说不，市场可能转冷
- 你以为会融到的钱可能永远不会来

**恶性循环**：
- Default dead → 需要融资 → 分心 → 产品停滞 → 更难融资

### 2. 为什么创始人不知道自己的状态

**认知偏差**：
- 乐观地高估未来增长
- 低估费用增长
- 把"可能的融资"当作"确定的资金"

**选择性忽视**：
- 这个问题的答案可能令人不安
- 不问就不用面对

### 3. 如何从 Default Dead 变成 Default Alive

**两条路径**：
1. **增加收入**：加速增长，更快接近盈利
2. **减少支出**：降低费用，延长跑道

**Graham 的建议**：通常先减支出
- 更可控
- 立即见效
- 给你时间去改善增长

### 4. 招聘是最大的杀手

**为什么**：
- 招人是最大的支出增加
- 但创始人把招人当作"进步"
- 实际上，过早招人是从 default alive 变成 default dead 的最常见原因

**建议**：
- 每次招人前问："如果不招，公司会死吗？"
- 如果答案是否，就不要招

## 实践建议

### 立即行动
1. **做计算**：现在就算出你是哪种状态
2. **诚实面对**：不要用乐观假设欺骗自己
3. **做决定**：如果是 default dead，立即采取行动

### 保持 Default Alive
1. **谨慎招人**：每个招聘都要慎重考虑
2. **控制支出**：不必要的支出直接砍掉
3. **专注增长**：而不是融资

### 如果是 Default Dead
1. **不要恐慌**：但要立即行动
2. **优先削减开支**：买时间
3. **加速产品迭代**：尽快找到 PMF
4. **认真考虑融资**：但不要依赖它

## 金句摘录

> "If you're default dead, you need to change something or raise more money, or you'll die."
> 
> 如果你是 default dead，你需要改变一些东西或筹集更多资金，否则你会死。

> "The startling thing is how often the founders themselves don't know whether they're default alive or default dead."
> 
> 令人震惊的是，创始人自己往往不知道他们是 default alive 还是 default dead。
