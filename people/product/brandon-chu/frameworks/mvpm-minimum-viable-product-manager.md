# MVPM: Minimum Viable Product Manager

**来源**: [Black Box of PM](https://blackboxofpm.com/mvpm-minimum-viable-product-manager-e1aeb8dd421)

---

## 框架概述

MVPM (Minimum Viable Product Manager) 定义了成为有效通用产品经理所需的最小技能集。这个框架帮助 PM 理解需要掌握的核心能力交集。

> "It defines a set of skills or knowledge that are useful to be an effective generalist product manager, one who can work on almost any problem."

---

## 核心模型

产品管理位于三个核心领域的交集：

```
        技术 (Tech)
           /\
          /  \
         /    \
        /  PM  \
       /________\
   商业 (Biz)   用户 (UX)
```

### 1. 技术知识 (The Stack)

PM 需要理解的技术基础：

- **系统架构** (System Architecture)
  - 前端/后端分离
  - 服务间通信
  - 数据流向

- **数据模型与 API** (Data Model and APIs)
  - 数据库结构理解
  - API 设计原则
  - 数据关系

- **不需要成为工程师**
  - 但要能与工程师有效沟通
  - 理解技术决策的影响
  - 评估技术方案的可行性

### 2. 影响力建模 (Modelling Impact)

理解决策如何影响业务：

- **量化框架**
  - 用户获取成本 (CAC)
  - 生命周期价值 (LTV)
  - 转化率分析

- **优先级决策**
  - 影响力 vs 努力度
  - 机会成本考量
  - 资源分配

### 3. 收集与综合信息 (Gather & Synthesize)

获取正确信息的能力：

- **用户研究**
  - 定性与定量结合
  - 用户访谈技巧
  - 数据分析能力

- **市场洞察**
  - 竞品分析
  - 行业趋势
  - 机会识别

## 实施指南

### 对于新 PM

1. **技术学习**
   - 学习基础编程概念
   - 理解产品技术栈
   - 与工程师建立信任

2. **商业思维**
   - 学习公司商业模式
   - 理解核心指标
   - 建立财务敏感度

3. **用户同理心**
   - 定期与用户交流
   - 使用自己的产品
   - 建立反馈循环

### 对于资深 PM

- 在已有强项基础上补足短板
- 指导新 PM 建立全面能力
- 深化某一领域的专业度

## 关键洞察

> "You don't need to be technical to be a PM, but you need to understand technology."

> "The best PMs can context-switch between business, tech, and user perspectives seamlessly."

---

## 应用场景

| 场景 | MVPM 应用 |
|------|----------|
| 面试准备 | 评估自己在三个维度的能力 |
| 职业规划 | 识别需要提升的领域 |
| 团队建设 | 确保团队覆盖所有维度 |
| 项目决策 | 平衡技术、商业、用户视角 |
