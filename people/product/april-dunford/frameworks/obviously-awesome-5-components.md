# Obviously Awesome — 产品定位 5 大核心组件

> "Positioning defines how your product is a leader at delivering something that a well-defined set of customers cares a lot about."
> — April Dunford

## 框架概述

April Dunford 在其畅销书《Obviously Awesome》中提出了产品定位的 5 个核心组件框架。这个框架的核心洞察是：**定位组件之间相互关联，必须按特定顺序逐一解锁**。

## 5 大核心组件

### 1. Competitive Alternatives（竞争替代方案）

**定义**: 如果你的产品不存在，客户会选择什么？

**关键点**:
- 不仅仅是直接竞争对手
- 包括 "什么都不做"、"手动处理"、Excel 表格等
- 代表客户心智中的参照物

**为什么重要**:
> "If there were no alternatives, we wouldn't need positioning at all because customers would simply buy what we're selling."

**实践建议**:
- 直接询问客户："在使用我们之前，你是怎么解决这个问题的？"
- 关注功能和能力的真正竞争，而非品牌竞争

---

### 2. Unique Attributes（独特能力/属性）

**定义**: 相比竞争替代方案，你的产品有哪些独特的功能或能力？

**关键点**:
- 必须是可验证的、客观的特性
- 是差异化的基础
- 可以是功能、架构、业务模式、专有知识等

**示例**:
- 专利技术
- 独特的数据资产
- 特定的集成能力
- 架构优势（如云原生 vs 本地部署）

**注意**:
> "Features only enable value — they are not the value itself."

---

### 3. Value（价值）

**定义**: 这些独特能力为客户带来什么实际价值？

**关键点**:
- 从独特属性推导而来
- 必须是客户真正关心的价值
- 区分功能（你有什么）和价值（客户得到什么）

**价值类型**:
- 节省时间/成本
- 增加收入
- 降低风险
- 提升体验

**核心公式**:
```
Unique Attributes → Value (for customers)
功能/能力 → 客户获得的实际好处
```

---

### 4. Target Market Characteristics（目标市场特征）

**定义**: 哪些客户最关心你提供的独特价值？

**关键点**:
- 不是所有客户都会同等重视你的价值
- 识别"最佳拟合客户"（Best-fit customers）
- 关注客户特征，而非人口统计

**识别方法**:
- 哪些客户购买决策最快？
- 哪些客户愿意支付更高价格？
- 哪些客户成为最强的推荐者？

**常见特征维度**:
- 公司规模
- 行业
- 技术栈
- 团队结构
- 紧迫度

---

### 5. Market Category（市场类别）

**定义**: 你的产品属于什么市场类别？

**关键点**:
- 类别是客户理解产品的心智捷径
- 影响客户对功能、定价、竞争对手的预期
- 可以选择现有类别或创建新类别

**三种策略**:

| 策略 | 适用场景 | 风险 |
|------|----------|------|
| **进入现有类别** | 产品明显更优 | 直接竞争 |
| **细分现有类别** | 有明确的目标细分市场 | 市场规模有限 |
| **创建新类别** | 产品无法用现有类别描述 | 教育成本高 |

**April 的建议**:
> "Don't create a new category unless you have no other choice. It's expensive and time-consuming to educate the market."

## 组件间的关系

```
┌─────────────────────────────────────────────────────────────┐
│                    5 Components Flow                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ① Competitive     ② Unique         ③ Value               │
│     Alternatives  →   Attributes   →   (for customers)     │
│                                                             │
│         │                                   │               │
│         │                                   ▼               │
│         │              ┌────────────────────────────────┐   │
│         │              │  ④ Target Market Characteristics│  │
│         │              │     (Who cares most?)          │   │
│         │              └────────────────────────────────┘   │
│         │                                   │               │
│         ▼                                   ▼               │
│  ┌──────────────────────────────────────────────────────┐   │
│  │           ⑤ Market Category                         │   │
│  │              (Context for understanding)            │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**顺序的重要性**:
> "The components build on each other. You can't determine your unique attributes without first understanding your competitive alternatives."

## 实践检查清单

- [ ] 明确识别了 3-5 个真正的竞争替代方案
- [ ] 列出了相对于替代方案的独特能力
- [ ] 将每个独特能力转化为客户可感知的价值
- [ ] 定义了最关心这些价值的客户特征
- [ ] 选择了能引发正确联想的市场类别
