# Positioning Canvas — 定位画布模板

> "This product positioning template was built for you. Deliberate positioning puts your strengths front and centre."
> — April Dunford

## 框架概述

Positioning Canvas 是 April Dunford 设计的**定位工作表**，帮助团队系统性地完成产品定位的 5 个核心组件。这个画布是 10 步定位流程的核心输出文档。

---

## 完整 Positioning Canvas 模板

```markdown
## [产品名称] Positioning Canvas

### 1. Competitive Alternatives（竞争替代方案）
**如果我们的产品不存在，客户会怎么做？**

| 替代方案 | 为什么客户选择它 |
|----------|----------------|
| [替代方案 A] | [原因] |
| [替代方案 B] | [原因] |
| [替代方案 C] | [原因] |

**关键问题**:
- 在使用我们之前，客户是如何解决这个问题的？
- 不仅仅是直接竞争对手——包括 Excel、手动流程、"什么都不做"

---

### 2. Unique Attributes（独特属性）
**相比这些替代方案，我们有什么独特能力？**

| 能力/特性 | 我们 | 替代方案 A | 替代方案 B |
|-----------|------|-----------|-----------|
| 特性 1 | ✓ | ✗ | ✗ |
| 特性 2 | ✓ | ✓ | ✗ |
| 特性 3 | ✓ | ✗ | ✓ |

**筛选标准**:
- 只保留竞争对手**没有**的特性
- 必须是可验证的、客观的
- 不是"更好"，而是"不同"

---

### 3. Value（价值）
**这些独特属性为客户带来什么实际价值？**

**转化公式**:
```
[独特属性] → 所以客户可以 → [价值]

例如：
"实时数据同步" → 所以客户可以 → "做出更快的决策"
```

**价值类型**:
- [ ] 效率价值（节省时间/金钱）
- [ ] 增长价值（增加收入/客户）
- [ ] 体验价值（更简单/更愉悦）
- [ ] 风险价值（降低风险/合规）

**价值主题**:
| 独特属性 | 客户价值 | 重要性 (1-10) |
|-----------|----------|---------------|
| [属性 1] | [价值] | [评分] |
| [属性 2] | [价值] | [评分] |

---

### 4. Target Market Characteristics（目标市场特征）
**哪些客户最关心这些价值？**

**最佳客户画像**:
| 特征 | 描述 |
|------|------|
| 公司规模 | [50-200人] |
| 行业 | [SaaS/B2B] |
| 技术栈 | [已有 CRM] |
| 组织成熟度 | [已建立流程] |
| 紧迫度触发因素 | [快速扩张中] |

**最佳客户特征**:
- [ ] 购买周期最短
- [ ] 价格敏感度最低
- [ ] 续约率最高
- [ ] 推荐意愿最强

---

### 5. Market Category（市场类别）
**我们的产品属于什么市场类别？**

**候选类别评估**:
| 类别 | 优势 | 风险 | 客户认知匹配度 |
|------|------|------|---------------|
| [类别 1] | | | /10 |
| [类别 2] | | | /10 |

**选择策略**:
- [ ] **Head-to-Head**: 在成熟类别中直接挑战领导者
- [ ] **Big Fish, Small Pond**: 在细分市场成为领导者
- [ ] **Create New Category**: 创造全新市场类别

**最终选择**: [你的市场类别]

**理由**: [为什么选择这个类别]

## 定位声明（Positioning Statement）

```markdown
For [目标客户]
Who [关键痛点/需求]
[产品名] is a [市场类别]
That [核心价值主张]
Unlike [主要替代方案]
We [关键差异化]
```

**示例**:
```
For B2B SaaS marketing teams
Who struggle with inconsistent brand positioning
Obviously Awesome is a positioning methodology
That provides a systematic approach to differentiation
Unlike generic marketing frameworks
We focus specifically on technology products with a proven 10-step process
```

## 使用指南

### 完成时间
- **工作坊**: 2-4 小时
- **准备阶段**: 1-2 周（客户研究 + 组建团队）
- **文档化**: 1 周

### 参与角色
| 角色 | 负责内容 |
|------|----------|
| 创始人/CEO | 决策、最终确认 |
| 产品负责人 | 产品能力、独特属性 |
| 销售负责人 | 客户洞察、竞争替代方案 |
| 市场负责人 | 价值传递、市场类别 |
| 客户成功负责人 | 客户反馈、价值验证 |

### 填写顺序
1. **Competitive Alternatives** → 从客户视角出发
2. **Unique Attributes** → 对比分析
3. **Value** → 转化为客户价值
4. **Target Market** → 识别最佳客户
5. **Market Category** → 选择正确类别
6. **Positioning Statement** → 整合为清晰声明

### 验证检查
完成 Canvas 后，询问以下问题：
- [ ] 我们的差异化是否清晰？
- [ ] 定位是否对目标客户"显而易见"？
- [ ] 团队是否全员理解并买入？
- [ ] 销售能否用这个定位讲述故事？
- [ ] 市场材料是否与此定位一致？

## 常见错误

### 1. 跳过竞争替代方案
> "If there were no alternatives, we wouldn't need positioning at all because customers would simply buy what we're selling."
>
> **正确**: 从客户现有解决方案开始，而非产品功能

### 2. 列出过多"独特"属性
> "Features only enable value — they are not the value itself."
>
> **正确**: 聚焦真正独特且有价值的属性

### 3. 混淆功能和价值
- ❌ "我们有 AI 功能"
- ✅ "AI 功能让你每天节省 2 小时"
>
> **正确**: 功能是你有什么，价值是客户得到什么

### 4. 忽视目标市场特征
> "Not all customers will care equally about your value."
>
> **正确**: 定义最关心你价值的客户特征

### 5. 选择错误的市场类别
> "Don't create a new category unless you have no other choice."
>
> **正确**: 选择能让价值"显而易见"的类别
