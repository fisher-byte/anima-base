# Beating the Averages

> **发表时间**: 2001年4月（2003年修订）
> **来源**: paulgraham.com（源自2001年Franz开发者研讨会演讲）
> **重要性**: Paul Graham 关于技术选择作为竞争优势的经典文章

## 核心论点

技术选择可以成为创业公司的战略优势。选择更强大的编程语言让 Viaweb 在竞争中脱颖而出。

## Viaweb 的故事

1995年夏天，Paul Graham 和 Robert Morris 创办了 Viaweb：
- 目标：让用户能够构建在线商店
- 创新：第一个基于 Web 的应用程序
- 选择：使用 Lisp 作为主要开发语言

### 为什么选择 Lisp

当时的情况：
- 大多数竞争对手使用 C++、Perl 或 PHP
- Lisp 被认为是"学术语言"
- 但 Lisp 的表达能力远超其他语言

**结果**：
- Viaweb 用更小的团队做出了更强大的产品
- 最终以约 $49M 卖给雅虎

## 编程语言的差异

### "Blub 悖论"

Paul Graham 提出了著名的 "Blub Paradox"：

假设存在一种假想的语言叫 "Blub"，处于语言能力的中间位置。

**Blub 程序员的视角**：
- **向下看**：能看到更低级语言的局限
- **向上看**：看不到更高级语言的优势

> "当你向下看时，你知道自己在向下看。但当你向上看时，你不知道自己在向上看。"

### 语言能力的连续谱

语言按表达能力排列：
```
低 ←————————————————→ 高
机器码 → 汇编 → C → Java → Python → Lisp
```

关键洞察：
- 更强大的语言让你用更少的代码做更多的事
- 代码少 = 开发快 = 迭代快 = 竞争优势

## 技术选择作为竞争优势

### 秘密武器

> "如果其他公司不愿意使用 Lisp，那就是我的竞争优势。"

**策略**：
- 选择对手不敢用的技术
- 用技术优势弥补资源劣势
- 保持技术领先

### 为什么大公司不这样做

大公司倾向于使用"安全"的技术：
- 容易招人
- 管理层能理解
- 符合行业标准

**这创造了机会**：
- 创业公司可以使用更强大的技术
- 用技术优势对抗资源优势

## 如何选择技术

### 评估标准

1. **表达能力**：能否用更少的代码实现功能
2. **迭代速度**：能否快速修改和测试
3. **团队能力**：团队是否能发挥语言的优势
4. **生态系统**：是否有足够的库和工具

### 警告

> "技术选择不能弥补糟糕的想法或执行。"

技术只是优势之一，不是唯一因素。

## 对今天的启示

虽然 Lisp 可能不是每个项目的最佳选择，但核心原则依然有效：

1. **质疑行业标准**：最流行的不一定最好
2. **寻找杠杆**：找到让小团队产生大影响的方式
3. **敢于不同**：竞争对手不敢做的事可能是机会

## 核心语录
