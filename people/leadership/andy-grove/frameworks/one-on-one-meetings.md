# One-on-One Meetings - 一对一会议框架

Andy Grove将一对一会议称为"管理工作的预防性医学",是建立和维持上下级关系的主要方式。

---

## 核心定义

> **"The one-on-one is the principal way your business relationship is maintained."**
>
> 一对一会议是维持你们业务关系的主要方式。

> **"A one-on-one is a free-form meeting between a supervisor and a subordinate. Its main purposes are mutual teaching and exchange of information."**
>
> 一对一是上级和下属之间的自由形式会议。其主要目的是相互教学和信息交换。

---

## 框架说明

### 1. 会议的核心目的

**不是:**
- ❌ 上级向下级分配任务
- ❌ 上级检查下属工作进度
- ❌ 上级批评下属的表现

**而是:**
- ✅ **下属的会议** - 由下属设置议程
- ✅ **信息交换** - 双向沟通和学习
- ✅ **问题解决** - 及早发现和处理问题
- ✅ **关系建立** - 建立信任和理解

### 2. 会议结构

```
会议组成 (60分钟标准)
├─ 下属的议题 (40分钟)
│   ├─ 项目进展更新
│   ├─ 遇到的障碍
│   ├─ 需要的支持
│   └─ 未来计划
├─ 上级的议题 (15分钟)
│   ├─ 公司战略更新
│   ├─ 期望澄清
│   └─ 反馈和指导
└─ 行动项确认 (5分钟)
```

### 3. Grove的关键原则

**原则1: 这是下属的会议**
```
下属设置议程
下属准备主题
下属引导讨论
上级提供支持
```

**原则2: 固定且神圣**
```
定期举行 (每周或每两周)
不能轻易取消
提前安排,写入日历
成为习惯和节奏
```

**原则3: 预防性医学**
```
及早发现问题
在小问题变大前解决
建立信任关系
避免突发危机
```

## 应用场景

### 场景1: 标准一对一流程

**会前准备 (下属):**
```markdown
## 1-on-1 Preparation - [Date]

### 我的更新
- 项目A: [状态/进展/下一步]
- 项目B: [状态/进展/下一步]
- 其他工作: [简要说明]

### 我遇到的障碍
1. [具体障碍] - 需要: [支持类型]
2. [具体障碍] - 需要: [支持类型]

### 我想讨论的主题
1. [主题1] - 为什么重要
2. [主题2] - 为什么重要
3. [主题3] - 为什么重要

### 我的问题
- [问题1]
- [问题2]

### 未来计划
- 下周: [计划]
- 下月: [重点]
```

**会议记录模板:**
```markdown
# 1-on-1 Notes - [Manager] & [Report] - [Date]

## 讨论的主题
### 1. [主题]
- 讨论内容: ___
- 关键洞察: ___
- 决定: ___

### 2. [主题]
- 讨论内容: ___
- 关键洞察: ___
- 决定: ___

## 反馈
- 做得好的: ___
- 可以改进的: ___

## 行动项
- [ ] [Action] - Owner: ___ - Due: ___
- [ ] [Action] - Owner: ___ - Due: ___

## 下次会议
- 日期: ___
- 准备的主题: ___
```

### 场景2: 不同类型的一对一

**A. 新员工一对一 (前3个月,每周)**
```
关注点:
├─ 入职进展
├─ 文化适应
├─ 关系建立
├─ 早期反馈
└─ 期望澄清

频率: 每周1小时
```

**B. 稳定员工一对一 (每两周)**
```
关注点:
├─ 项目进展
├─ 职业发展
├─ 战略对齐
├─ 问题预防
└─ 知识分享

频率: 每两周45-60分钟
```

**C. 高级员工一对一 (每月)**
```
关注点:
├─ 战略讨论
├─ 组织问题
├─ 领导力发展
├─ 大局思考
└─ 教练辅导

频率: 每月60-90分钟
```

**D. 危机一对一 (按需)**
```
关注点:
├─ 绩效问题
├─ 关系冲突
├─ 重大挑战
├─ 职业困惑
└─ 紧急支持

频率: 按需,可能更频繁
```

### 场景3: 具体讨论技巧

**提问技巧 (上级):**

```markdown
## 开放式问题 (鼓励思考)
- "你对这个项目的进展有什么想法?"
- "你觉得团队现在最大的挑战是什么?"
- "如果你是我,你会怎么处理这个情况?"

## 探索式问题 (深入理解)
- "能详细说说吗?"
- "为什么你觉得这是个问题?"
- "还有什么其他因素?"

## 教练式问题 (促进成长)
- "你尝试过什么方法?"
- "你认为有哪些选项?"
- "你需要什么来做出这个决定?"

## 避免的问题
- ❌ "你完成X了吗?" (太直接)
- ❌ "为什么没做?" (指责性)
- ❌ "你应该..." (命令式)
```

## 实施步骤

### Step 1: 建立节奏 (Week 1)

**设置框架:**
```markdown
## 一对一会议启动

### 与团队成员沟通
"我想和你建立定期的一对一会议:
- 频率: [每周/每两周]
- 时长: [45-60分钟]
- 目的: 这是你的会议,讨论你的优先事项
- 格式: 你准备议程,我提供支持"

### 日历设置
- [ ] 选择固定时间
- [ ] 设置循环会议
- [ ] 预留缓冲时间
- [ ] 不能与其他会议冲突
```

### Step 2: 建立模板 (Week 2)

**共享文档:**
```
工具选择:
├─ Google Doc (推荐)
├─ Notion
├─ OneNote
└─ 专用工具 (Fellows, Lattice等)

文档结构:
├─ 长期主题 (持续追踪)
├─ 本次议程
├─ 会议笔记
└─ 行动项追踪
```

### Step 3: 培养习惯 (Month 1-3)

**早期重点:**

```
第1-4次会议:
├─ 建立信任
├─ 了解工作风格
├─ 澄清期望
└─ 设定节奏

第5-8次会议:
├─ 深入项目
├─ 讨论挑战
├─ 提供反馈
└─ 支持成长

第9-12次会议:
├─ 战略对齐
├─ 职业发展
├─ 预防问题
└─ 知识分享
```

### Step 4: 持续优化 (Ongoing)

**定期检查 (每季度):**

```markdown
## 一对一效果检查

### 问自己
- 会议是否有价值?
- 是否发现了问题?
- 关系是否改善?
- 需要调整什么?

### 问团队成员
- "这些会议对你有帮助吗?"
- "我们应该更多讨论什么?"
- "我们应该少讨论什么?"
- "格式需要调整吗?"
```

## 常见误区

### ❌ 误区1: "我太忙了,没时间开一对一"

**Grove的回答:**
> "If you don't have time for 1-on-1s, you don't have time to be a manager."

**现实:**
- 一对一是最高ROI的管理活动
- 预防问题比救火更高效
- 30分钟一对一 >> 3小时救火

### ❌ 误区2: "一对一就是状态更新"

**问题:**
- 变成项目管理会议
- 错失深入讨论机会
- 浪费双方时间

**正确做法:**
```
状态更新 → 异步完成 (Slack/邮件)
一对一 → 讨论复杂问题、战略、成长
```

### ❌ 误区3: "只在有问题时才开"

**危险:**
- 一对一变成"被召见"
- 失去预防性功能
- 破坏信任关系

**正确认知:**
- 定期会议建立安全感
- 无事可谈也是信号
- 持续对话预防危机

### ❌ 误区4: "我来设置所有议程"

**问题:**
- 变成上级的会议
- 下属被动接受
- 失去双向沟通

**Grove原则:**
> "The meeting must be the subordinate's meeting, with his or her agenda."

## 高级技巧

### 技巧1: 职业发展讨论

**每月深入讨论:**

```markdown
## 职业发展一对一

### 当前技能评估
- 你觉得自己在哪些方面很强?
- 哪些方面想要提升?
- 最享受什么类型的工作?

### 职业目标
- 6个月后想达到什么状态?
- 1年后呢?
- 3年后呢?

### 发展计划
- 需要学习什么新技能?
- 需要什么类型的项目?
- 我可以如何支持你?

### 行动项
- [ ] [具体发展活动]
- [ ] [具体发展活动]
```

### 技巧2: 360度反馈

**定期(每季度)讨论:**

```markdown
## 相互反馈

### 我给你的反馈
- 做得很好的: ___
- 可以改进的: ___
- 具体例子: ___

### 你给我的反馈
- 我作为管理者做得好的: ___
- 我可以改进的: ___
- 你需要我更多/更少做什么: ___
```

### 技巧3: "Skip Level" 一对一

**定期与下下级见面:**

```
目的:
├─ 了解更广泛的团队动态
├─ 发现中层管理问题
├─ 建立更广泛的关系
└─ 发现新兴人才

频率: 每季度
注意: 不是绕过直接上级,而是补充视角
```
