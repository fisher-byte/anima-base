# 内容质量指南

本文档定义了Anima Base项目的内容质量标准和验证流程。

---

## 🎯 内容定位

### 项目性质
Anima Base是一个**知识策展项目**，而非档案馆或百科全书。

- ✅ **我们提供**: 核心洞察、框架总结、精华提炼
- ❌ **我们不是**: 完整转录、逐字记录、学术论文

### 目标用户
- 产品经理、营销人员、增长专家
- 寻求快速学习行业智慧的从业者
- 需要框架和方法论指导的团队

---

## ✅ 质量标准

### 1. 真实性 (Authenticity)

#### 必须符合
- [ ] 人物真实存在且职业经历可验证
- [ ] 核心理论/框架在公开资料中可找到
- [ ] 关键事件（如Airbnb麦片盒故事）经网络验证
- [ ] 代表性作品真实存在（书籍、播客等）

#### 禁止内容
- ❌ 编造的框架或方法论（如虚假的"11个问题"）
- ❌ 无法验证的具体数字或日期
- ❌ 捏造的对话或引语
- ❌ 混淆不同概念的内容

### 2. 可追溯性 (Traceability)

#### 必须提供
- [ ] 在YAML frontmatter中列出主要来源
- [ ] 为核心框架/理论添加来源链接
- [ ] 重要语录标注出处（优先级高）
- [ ] **新增：所有内容必须有 `original_file` 或 `original_urls` 字段**
- [ ] **新增：原文文件已下载到 `files/` 或 `media/` 目录**

#### 原文保存要求（强制执行）

**根据 FILE_STORAGE_POLICY.md，必须执行双轨制：**

1. **文章内容**
   - [ ] 保存 HTML → `files/articles/{person}/{date}-title.html`
   - [ ] 保存 PDF（如有）→ `files/pdf/{person}/articles/{date}-title.pdf`
   - [ ] 索引文件包含 `original_file` 字段指向原文

2. **播客内容**
   - [ ] 下载 MP3 → `media/audio/{person}/podcast/{date}-show-name.mp3`
   - [ ] 保存转录 TXT → `media/audio/{person}/podcast/{date}-show-name-transcript.txt`
   - [ ] 索引文件包含 `original_urls` 字段（youtube, spotify等）

3. **书籍内容**
   - [ ] 保存 EPUB → `media/ebook/{title}.epub`
   - [ ] 保存 PDF → `files/pdf/{person}/books/{title}.pdf`

#### 禁止行为
- [ ] 禁止只保存摘要，必须保存原文文件
- [ ] 禁止使用2-3KB的索引文件替代原文
- [ ] 原文含量需达到 90% 以上（原文文件大小与索引文件大小比）

#### 来源优先级
1. **高优先级** (必须有来源):
   - 核心理论和框架
   - 代表性语录
   - 关键数据和时间

2. **中优先级** (建议有来源):
   - 职业经历细节
   - 重要事件描述
   - 方法论说明

3. **低优先级** (可以总结):
   - 背景信息
   - 一般性描述
   - 风格特点

### 3. 时效性 (Timeliness)

#### 日期要求
- [ ] 所有日期必须早于或等于今天
- [ ] 采集日期 (date_collected) 必须合理
- [ ] 发布日期 (published_date) 可通过网络验证

#### 内容更新
- [ ] 人物的current_role反映最新状态
- [ ] 重大职业变动需要更新
- [ ] 过时内容添加更新时间说明

### 4. 完整性 (Completeness)

#### Profile必需元素
```yaml
---
person: 人物姓名
role: 当前职位
category: 领域分类
tags: [标签列表]
sources: [来源列表]
verification_status: reviewed/needs-review/unverified
verification_date: YYYY-MM-DD
content_type: curated_summary/full_transcript/verified
---
```

#### 内容必需部分
- [ ] 基本信息（姓名、职位、领域）
- [ ] 职业经历概述
- [ ] 核心理念/代表性观点
- [ ] 推荐资源（书籍、播客、文章等）

### 5. 结构化 (Structure)

#### 格式规范
- [ ] 使用Markdown格式
- [ ] 清晰的标题层级（H1-H3）
- [ ] 列表和表格适当使用
- [ ] 代码块标注语言

#### 命名规范
- 文件名: `lowercase-with-hyphens.md`
- 目录名: `lowercase-folder`
- 日期格式: `YYYYMMDD` 或 `YYYY-MM-DD`

## 🔍 验证流程

### 新增内容

#### Step 1: 创建草稿
1. 使用模板创建文件
2. 标记 `verification_status: needs-review`
3. 填写基本信息和来源

#### Step 2: 内容编写
1. 基于可靠来源编写内容
2. 为关键观点添加来源链接
3. 避免编造细节

#### Step 3: 自我检查
- [ ] 关键信息通过网络搜索验证
- [ ] 所有重要语录有来源
- [ ] 日期逻辑合理
- [ ] 无明显错误

#### Step 4: 提交审查
1. 提交Pull Request
2. 在PR描述中列出主要来源
3. 说明验证过程

#### Step 5: 审查通过
1. 维护者审查内容
2. 修改 `verification_status: reviewed`
3. 添加 `verification_date`
4. 合并到主分支

### 定期审查

#### 季度审查 (每3个月)
- [ ] 检查人物职位是否有变动
- [ ] 验证来源链接有效性
- [ ] 更新过时内容
- [ ] 标记需要重新验证的内容

#### 年度审查 (每年)
- [ ] 全面审查所有profile
- [ ] 更新统计数据
- [ ] 移除长期过时的内容
- [ ] 补充重要新内容

## ⚠️ 常见问题防范

### AI生成内容陷阱

#### 易产生幻觉的场景
1. **具体数字和列表**
   - ❌ "7个步骤"、"11个问题"等可能编造
   - ✅ 如需列举，必须有来源链接

2. **混淆相似概念**
   - ❌ "11星vs11问题"的混淆
   - ✅ 仔细区分相似名称的概念

3. **编造细节**
   - ❌ 虚构的对话、会议、决策
   - ✅ 只引用可验证的公开信息

4. **夸张表述**
   - ❌ "史上最好"、"唯一"等绝对化
   - ✅ 使用客观、可验证的描述

### 检查清单

在提交内容前，问自己：

- [ ] 这个框架/方法论在Google能搜到吗？
- [ ] 这个语录能找到原文链接吗？
- [ ] 这个数字/日期通过搜索验证了吗？
- [ ] 如果我删掉这段，会影响核心信息吗？（冗余检查）
- [ ] 这个描述是否过于绝对化？

## 📋 验证状态说明

### verification_status字段

| 状态 | 说明 | 适用场景 |
|------|------|----------|
| `reviewed` | 已审查 | 内容经过人工审查，关键信息已验证 |
| `needs-review` | 待审查 | 新创建的内容，等待审查 |
| `unverified` | 未验证 | 历史内容，尚未经过系统验证 |
| `outdated` | 已过时 | 内容已过时，需要更新 |

### content_type字段

| 类型 | 说明 | 示例 |
|------|------|------|
| `curated_summary` | 策展摘要 | 大部分profile文件 |
| `full_transcript` | 完整转录 | 播客逐字稿 |
| `verified` | 完全验证 | 每句话都有来源的内容 |
| `draft` | 草稿 | 正在编写中 |

## 🤝 贡献指南

### 如何贡献

1. **报告问题**
   - 发现幻觉内容 → 提交Issue
   - 说明具体位置和问题
   - 提供验证过程

2. **修复内容**
   - Fork仓库
   - 创建修复分支
   - 提交PR并说明改动

3. **添加来源**
   - 为现有内容补充来源链接
   - 优先处理核心框架和语录
   - 在PR中列出添加的来源

4. **验证内容**
   - 帮助验证`needs-review`的内容
   - 在Issue中分享验证结果
   - 参与内容质量讨论

### Pull Request要求

```markdown
## PR标题格式
- feat: 添加新内容
- fix: 修复错误
- docs: 更新文档
- verify: 内容验证

## PR描述模板
### 改动说明
简要描述改动内容

### 验证过程
- 搜索关键词: xxx
- 找到来源: [链接]
- 验证结果: ✅/❌

### 检查清单
- [ ] 关键信息已验证
- [ ] 添加了来源链接
- [ ] 更新了verification_status
- [ ] 检查了日期逻辑
```
